# pcluster-install-with-kiro

Kiro와 한국어로 대화하면서 AWS ParallelCluster(HPC 클러스터)를 쉽게 배포할 수 있는 프로젝트입니다.

## 왜 만들었나요?

대학 연구실에서 HPC 클러스터가 필요할 때, AWS 콘솔을 직접 다루거나 복잡한 CLI 명령어를 익히는 건 연구자에게 부담입니다.

대학 연구실은 보통 on-premise 장비로 연구를 수행합니다. 하지만 대규모 시뮬레이션이나 딥러닝 학습처럼 자원이 부족해질 때가 있습니다. 이때 AWS에 HPC 클러스터를 만들어 **몇 시간이든, 며칠이든 필요한 만큼만 사용하고 삭제**할 수 있다면 이상적입니다.

문제는 FSx for Lustre 같은 고성능 스토리지는 **생성만 해두어도 시간당 비용이 발생**한다는 점입니다. 사용이 끝나면 바로 삭제해야 하는데, "다시 만들기 어려우면 삭제도 망설이게" 됩니다.

이 프로젝트는 그 문제를 해결합니다. Kiro에게 "시작"이라고 말하면, **복잡한 IT 명령어 없이도 간단한 선택만으로 약 30분 만에 네트워크부터 클러스터까지 구성할 수 있습니다.** 다시 만드는 데 부담이 없으니, 삭제도 망설일 필요가 없습니다.

> 💡 **필요할 때 만들고 → 쓰고 → 바로 삭제.** 이 사이클을 부담 없이 반복할 수 있도록 만들었습니다.

- AWS 용어나 Shell 명령어를 몰라도 됩니다
- 번호 선택만으로 클러스터를 구성할 수 있습니다
- 모든 안내가 한국어로 제공됩니다
- VPC 네트워크 생성부터 클러스터 배포까지 약 30분이면 완료됩니다

## 주요 특징

### 1. 프리셋 + 커스텀 이중 구조
- 입문용 / 전산 시뮬레이션 / AI·딥러닝 3종 프리셋으로 번호 선택만으로 바로 시작 가능
- 커스텀 모드에서는 OS, 인스턴스, 스토리지, EFA, DCV 등을 연구 목적에 맞게 세밀하게 설정 가능

### 2. 가용영역(AZ) 자동 검증
- 연산 인스턴스가 실제로 제공되는 가용영역만 필터링하여 선택지로 제시
- 배포 후 "해당 AZ에서 인스턴스를 사용할 수 없습니다" 같은 실패를 사전에 방지
- 용량 부족 발생 시에도 AZ 변경 → 네트워크 재생성 → 재배포 흐름을 자동 안내

### 3. 네트워크 자동 생성 / 기존 VPC 재사용
- 프리셋은 클러스터 전용 VPC + public/private 서브넷을 CloudFormation으로 자동 생성하여 네트워크 설계 부담을 제거
- 커스텀은 기존 VPC를 선택하거나 새로 생성할 수 있어 운영 중인 환경에도 유연하게 통합 가능

### 4. 템플릿 기반 설정으로 LLM 할루시네이션 최소화
- CloudFormation 템플릿과 ParallelCluster 설정 템플릿을 미리 정의
- LLM이 설정을 "생성"하는 것이 아니라, 검증된 템플릿의 변수를 "치환"하는 방식
- AI가 잘못된 파라미터나 구조를 만들어내는 위험을 구조적으로 차단

### 5. 몰라도 고를 수 있는 선택지
- 모든 단계에 용도별 추천 표시(⬅ 추천)와 구체적인 예시를 함께 제공
- 연구 분야별(CFD, 딥러닝, 바이오인포매틱스 등) 인스턴스 추천으로 AWS 지식 없이도 적절한 구성 가능

### 6. 생성부터 삭제까지 완전한 라이프사이클
- 클러스터 생성뿐 아니라, 삭제 시 네트워크 스택까지 단계별로 정리 안내
- "필요할 때 만들고 → 쓰고 → 바로 삭제" 사이클을 부담 없이 반복할 수 있도록 설계
- FSx for Lustre 등 유휴 비용이 발생하는 리소스도 망설임 없이 정리 가능

## 대상 사용자

- 대학 연구실 연구자 (본인 AWS 계정 사용)
- AWS/Shell 경험이 적은 비개발자

## 사전 준비

- AWS 계정 및 자격증명 설정 (`aws configure`)
- [Kiro CLI](https://kiro.dev) 설치
- Python 3.9 이상

## 사용 방법

1. 이 저장소를 클론합니다:
   ```bash
   git clone https://github.com/icn-univ/pcluster-install-with-kiro.git
   cd pcluster-install-with-kiro
   ```

2. Kiro CLI를 실행합니다:
   ```bash
   kiro-cli chat
   ```

3. pc-agent로 전환합니다:
   ```
   /agent swap pc-agent
   ```

4. "시작"이라고 입력하면 Kiro가 자동으로 클러스터 프리셋 안내를 시작합니다:
   ```
   1. 입문용 클러스터 — 코드 테스트, 소규모 병렬 계산
   2. 전산 시뮬레이션 클러스터 — FEM, CFD (ANSYS, OpenFOAM 등)
   3. AI/딥러닝 클러스터 — 모델 학습, 데이터 분석
   4. 🛠️ 커스텀 클러스터 — 직접 설정
   9. 🗑️ 기존 생성된 클러스터 삭제
   ```

5. 클러스터 이름을 입력하면 Kiro가 6단계로 자동 배포합니다:
   - AWS 계정 연결 확인
   - pcluster CLI 설치 확인
   - SSH 접속 키 확인
   - 네트워크 생성 (VPC/서브넷)
   - 클러스터 설정 파일 생성
   - 클러스터 배포

## 프리셋 클러스터

| # | 이름 | 용도 | 관리 서버 | 연산 서버 | 공유 저장소 |
|---|------|------|-----------|-----------|------------|
| 1 | 입문용 | 코드 테스트, 수업용 | t3.medium | c7i.large × 최대 4대 | EFS |
| 2 | 전산 시뮬레이션 | FEM, CFD | c7i.xlarge | c8i.48xlarge × 최대 8대 | FSx Lustre 1.2TB |
| 3 | AI/딥러닝 | 모델 학습, 데이터 분석 | c7i.xlarge | g6.xlarge × 최대 8대 | FSx Lustre 1.2TB |

## 프로젝트 구조

```
├── .kiro/agents/
│   └── pc-agent.json          # Kiro agent 설정 (실행 진입점)
├── .kiro/steering/
│   └── hpc-installer.md       # Kiro 동작 지침 (핵심 steering 파일)
├── sample-vpc-subnet.json     # VPC/서브넷 CloudFormation 템플릿
├── sample-hpc-cluster.yaml    # 클러스터 설정 템플릿 (변수 치환 방식)
└── README.md
```

- `sample-vpc-subnet.json` — 네트워크 생성에 사용되는 CloudFormation 템플릿. 새 VPC 생성 또는 기존 VPC 재사용을 지원합니다.
- `sample-hpc-cluster.yaml` — ParallelCluster 설정 템플릿. 프리셋/커스텀 선택에 따라 변수가 치환되어 최종 config.yaml이 생성됩니다.

## 커스텀 클러스터

4번(커스텀)을 선택하면 아래 항목을 직접 설정할 수 있습니다:

1. 클러스터 이름
2. 운영체제 선택
3. 관리 서버(Head Node) 인스턴스 타입
4. 연산 서버(Compute Node) 인스턴스 타입 — 용도별 추천 제공
5. 연산 서버 최대 수
6. 공유 스토리지 (EFS / FSx for Lustre)
7. 고속 네트워크 EFA (노드 간 병렬 통신)
8. 원격 데스크톱 DCV (웹 브라우저 GUI 접속)

## 클러스터 삭제

⚠️ 사용이 끝나면 반드시 클러스터를 삭제하세요. (비용 발생 방지)

```bash
pcluster delete-cluster --cluster-name <클러스터명> --region <리전>
```
